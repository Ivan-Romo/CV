  • Chess Project

    A project to create a digital version of chess using Unity and C# programming.



    In this project, I focused on implementing chess mechanics efficiently. The goal was to have a functional chess game that could later be enhanced with artificial intelligence. Therefore, position calculations needed to be as simple as possible, so that when adding an AI, the system could dedicate more power to applying the Minimax algorithm and spend less time on game calculations. Here are two interesting features of the code:


    I had the idea of implementing the board as a vector instead of a matrix. I am aware this is somewhat counterintuitive, but this way, I could move a rook by simply adding or subtracting one (vertically) or adding or subtracting 8 (horizontally). There is a photo explaining how this works.


    Additionally, I found that some projects used binary values for pieces. We started by assigning an integer value to each piece; for example, the white knight was 11, and the black knight was 19.


    11 in binary is 1011 and 19 is 10011. The first two bits of the number indicate the piece type, and the next three indicate its color.


    My code implements a similar approach, and I plan to leverage this further when implementing the AI.

    Together with a colleague, we developed a web page where you can upload image URLs, and it returns an image with all detected faces from that URL. This project arose from seeing that Azure provided us with student licenses, so we aimed to create a project using the tools it offered.

  • Shopping List App

    A shopping list app developed with Flutter, Dart, and Firebase



    I developed this app with the goal of learning Flutter. My initial intention was to use a MySQL database, but as I researched, I found there was a lot of compatibility between Flutter and Firebase. Since both technologies were new to me, I decided to try Firebase.


    The app allows each user to have two completely personal lists. One is for commonly purchased items, and the other is for the actual shopping list. Thus, a user who wants to quickly create a shopping list can view all the items they usually buy and add them to the shopping list.


    All user data is stored in the cloud. The app does not store any data locally.


  • Mobile Game

    I set out to create a hyper-casual mobile game, involving myself as much as possible in all aspects of development.



    This project started with several goals in mind: creating a game that could be launched on the Play Store, understanding and defining a specific target audience, learning how to use tools to analyze game performance on the Play Store, and learning to use AdMob.


    The first goal was the most realistic; I just needed to create a simple, coherent game. Therefore, I chose to make a hyper-casual game because what I wanted from this project was not to create a complex game but to learn about other aspects involved in game development.


    For targeting, I decided to get inspired by what hyper-casual game developers do, using Ketchapp as a reference. As I researched, I realized that targeting was crucial; every project decision had to be made with this in mind—from mechanics and aesthetics to the game name and interface complexity or the number of on-screen particles.


    Additionally, part of the development involved making the game intuitive. I conducted tests with people of various ages to see who responded best. The result was that, on average, without any indication, people needed about six attempts to understand how the game worked (which was too much), so I simplified it and provided visual cues.


    The game ultimately remained unfinished, but even though it wasn’t launched, I learned a great deal about game development.
